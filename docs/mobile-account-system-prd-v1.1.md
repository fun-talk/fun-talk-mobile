# 欧波开心学 Mobile 端账号体系与后台管理 PRD

版本：v1.1  
日期：2026-06-27  
适用项目：`fun-talk-mobile`  
前置版本：`mobile-account-system-prd.md`（Mobile v1.0）  
Web 基线：`../../PRD-v1.1.md`（Web 端 v1.1 账号体系与后台管理已完成）  
设计图目录：`../../web端网页注册登录后台3/`

## 1. 文档目的

Mobile v1.0 只复刻 Web 端学生/家庭登录、统一个人中心、多孩子绑定等学生与家长能力，明确不包含教师注册/登录、后台管理、学生编号生成、ID 导出、学校信息合并等教师/管理员能力。

Mobile v1.1 的目标是在 App 内承接 Web 端 v1.1 已完成的教师/管理员能力，并在原生端完整复刻 Web 端账号体系 UI、字段、交互、权限和业务规则，使老师和管理员可以不依赖 Web 浏览器完成账号创建、学生编号分发、学生账号管理、老师账号管理、学校信息合并和导出/打印操作。

本 PRD 是 Mobile v1.0 的扩展版，包含 v1.0 全部学生/家庭范围，并新增教师/管理员后台范围。

## 2. 产品定位

| 端 | 主要用户 | v1.1 定位 |
| --- | --- | --- |
| Web 端 | 老师、管理员、学生、家长 | 账号体系与后台管理基线，作为 Mobile UI 和业务规则的唯一对齐来源 |
| Mobile 端 v1.0 | 学生、家长 | 复刻学生/家庭登录、个人中心和绑定逻辑 |
| Mobile 端 v1.1 | 老师、管理员、学生、家长 | 复刻 Web v1.1 全账号体系，新增教师/管理员后台原生能力 |

Mobile v1.1 继续遵守隐私原则：学生登录和分发只使用系统生成的数字 ID、班级学号后 2 位、昵称、初始/临时密码；学生端不展示学校名、班级名、真实姓名、真实完整学号、身份证号等私人信息。

## 4. UI 复刻原则

Mobile v1.1 不重新设计账号后台，而是以 Web v1.1 为视觉和交互基线。

| 项目 | 要求 |
| --- | --- |
| 视觉基线 | 以 `../../web端网页注册登录后台3/` 中 11 张设计图和 `fun-talk-web/src/account/account.css` 为准 |
| 组件风格 | 复刻 Web 的品牌顶栏、狐狸头像、白色卡片、浅蓝统计卡、圆角输入框、主按钮、次按钮、标签、表格、弹窗 |
| 文案 | 页面标题、按钮文案、字段名、错误提示优先沿用 Web v1.1 |
| 字段 | 不新增会改变业务含义的字段；不删除 Web 已有字段 |
| 状态 | 加载中、空列表、导出中、打印准备中、合并中、待重置、已绑定、未绑定等状态必须完整呈现 |
| 权限 | 所有入口和操作按钮按服务端角色与权限展示；本地角色只用于展示，不作为鉴权依据 |
| 响应式 | 平板/横屏优先复刻 Web 双栏和表格；手机竖屏将双栏改为上下堆叠，表格可横向滚动或转为字段卡片，但字段顺序、操作和视觉层级必须与 Web 一致 |
| 图标与品牌 | 使用现有 Mobile 登录资产；品牌色、Logo、小狐狸元素与 Web 保持一致 |

验收时需要逐页对照 Web 截图，确认 UI 差异属于移动端空间限制导致的布局折叠，而不是信息架构或视觉语言变更。

## 7. 登录与角色路由

### 7.1 学生/家庭登录

沿用 Mobile v1.0 范围，继续对齐 Web v1.1：

| 模块 | 要求 |
| --- | --- |
| 家庭个人账号登录 | 微信登录、手机号验证码登录、手机号密码登录，首次登录自动创建家庭账号 |
| 学校账号登录 | 数字 ID + 密码，不支持昵称登录 |
| 协议勾选 | 未勾选《用户协议》《隐私协议》《儿童隐私政策》不能登录 |
| 忘记密码 | 输入数字 ID 后通知管理员和教学老师重置 |
| 登录后路由 | 学生/家庭账号进入课程首页，可进入统一个人中心 |

### 7.2 老师注册 / 登录入口

学生登录页顶部保留 Web 同款「老师注册 / 登录」入口。点击后进入老师认证页，页面头部、卡片宽度、模式切换按钮、输入框、协议勾选和按钮状态均复刻 Web。

### 7.3 老师登录

| 区域 | 字段 / 操作 | 规则 |
| --- | --- | --- |
| 页面头部 | Logo、小狐狸、标题「老师登录」、切换按钮「没有账号，去注册」 | 对齐 Web |
| 手机号 | 手机号输入框 | 必填 |
| 登录方式 Tab | 验证码登录 / 密码登录 | 与 Web 样式一致 |
| 验证码登录 | 验证码输入、获取验证码、倒计时 | 手机号为空不可获取 |
| 密码登录 | 登录密码、显示/隐藏密码 | 密码必填 |
| 协议 | 用户协议、隐私协议 | 未同意不可登录 |
| 登录成功 | 根据 `teacher.role` 跳转管理员或教学老师后台 | 管理员进入老师账号管理，普通老师进入学生账号管理 |

### 7.4 老师注册

| 字段 | 规则 |
| --- | --- |
| 手机号 | 必填，需验证码验证 |
| 学校名 | 必填，支持搜索选择或手动输入后由服务端归并 |
| 登录密码 | 必填，遵守 Web 密码强度规则 |
| 确认密码 | 必填，需与登录密码一致 |
| 姓名 / 显示名称 | 可选，未填写时按手机号脱敏展示 |
| 邮箱 | 可选，用于找回密码或后台联系 |
| 验证码 | 必填 |
| 协议勾选 | 未同意不可注册 |

注册成功后按 Web 规则创建老师账号；若为学校首个老师账号，由服务端判定是否具备管理员权限。

### 8.1 页面骨架

Mobile 后台使用原生页面复刻 Web `AccountLayout`：

| 区域 | 管理员 | 教学老师 |

手机竖屏中，导航可使用顶部横向 Tab、底部 Tab 或抽屉，但入口名称、可见性和顺序必须与 Web 端权限一致。平板/横屏优先使用侧边栏或顶部导航复刻 Web 后台。

| 组件 | 复刻要求 |
| --- | --- |

| 弹窗 | 半透明遮罩、居中卡片、标题、说明、主/次按钮与 Web 一致 |
| 输入弹窗 | 用于重置密码等操作，支持空值提交规则 |
| Toast | 成功、错误、提示三类反馈 |
| 加载骨架 | 列表加载时展示与 Web 等价的加载态 |

## 9. 学生编号生成

### 9.1 入口

管理员和具备创建权限的教学老师可从学生账号管理页进入「注册学生编号信息」。

### 9.2 单个创建

| 字段 | 规则 |
| --- | --- |
| 学校名 | 自动带入当前老师所属学校，只读 |
| 班级名 | 必填，例如「三年级 2 班」 |
| 班级学号后 2 位 | 必填，仅数字，自动补齐 2 位 |
| 学生初始密码 | 必填，遵守 Web 密码强度规则 |

提交按钮文案为「下一步，生成学生ID」。成功后进入学生编号生成结果页。

### 9.3 批量导入

| 项目 | 要求 |
| --- | --- |
| 入口 | 学生编号注册页右侧「Excel 批量导入」卡片 |
| 文件类型 | 优先支持 CSV；如接入原生文件选择器，可兼容 `.xlsx` 后由服务端解析 |
| 模板字段 | 学校名、班级名、班级学号的后 2 位、学生初始密码 |
| 校验 | 空文件、字段缺失、密码强度不合规、导入失败行均需提示 |
| 成功 | 显示成功数量和失败数量，至少展示首个生成结果并支持回到列表 |

### 9.4 生成结果页

| 区域 | 内容 |
| --- | --- |
| 标题 | 已生成的学生数字编号和密码 |
| 提示 | 请将以下信息交给学生，数字 ID 用于登录 |
| 统计卡 | 学校、班级、学生数字编号 |
| 结果表 | 数字 ID、班级学号后 2 位、初始密码、昵称状态 |
| 操作 | 导出 Excel、打印账号卡、返回学生列表 |

生成结果页必须支持 Web v1.1 新增的「导出 Excel」「打印账号卡」。

## 10. 管理员后台 - 学生账号管理

### 10.1 页面能力

| 区域 | 要求 |
| --- | --- |
| 标题 | 学生账号管理 |
| 顶部操作 | 单个创建、批量导入 CSV、导出 Excel、打印账号卡 |
| 班级筛选 | 默认全部班级，支持按班级过滤 |
| 待处理忘记密码申请 | 有申请时展示数字 ID、昵称、班级 |
| 学生列表 | 全校权限范围内学生 |

### 10.2 列表字段

| 字段 | 说明 |
| --- | --- |
| 数字 ID | 学生登录凭证；有待重置申请时显示「待重置」标签 |
| 昵称 | 未设置时灰色显示「未设置」 |
| 班级 ID 后 2 位 | 老师线下识别 |
| 密码 | 默认脱敏；重置后临时展示新临时密码 |
| 学校与家庭数据绑定 | 已绑定 / 未绑定 |
| 操作 | 重置密码、查看家庭数据 |

### 10.3 重置密码

管理员点击「重置」后弹出输入弹窗：

```text
标题：重置学生密码
说明：学生 <digital_id>，留空则随机生成新密码。
输入：新密码（可选）
操作：取消 / 重置
```

若输入新密码，需校验密码强度；若留空，由服务端生成临时密码。重置成功后列表中临时展示新密码，并清除待重置状态。

### 10.4 导出与打印

| 操作 | 规则 |
| --- | --- |
| 导出 Excel | 导出当前筛选范围内学生账号 |
| 打印账号卡 | 生成当前筛选范围账号卡 |
| 禁用状态 | 加载中、导出中、打印准备中、筛选结果为空时禁用 |
| 审计 | 服务端记录导出和打印任务 |

| 班级列表 | 班级名、所属老师、学生数量、创建时间 |
| 待合并记录 | 类型、相似名称、合并目标、影响学生、操作 |
| 合并记录 | 类型、合并前、合并后、影响老师、影响学生、时间 |

### 13.3 合并规则

| 规则 | 说明 |
| --- | --- |
| 管理员限定 | 仅管理员可执行学校/班级合并 |
| 二次确认 | 合并前弹窗展示影响范围 |
| 学生数字 ID 不变 | 合并不会影响学生登录 |
| 老师权限迁移 | 原班级权限迁移到合并后班级 |
| 学习数据不变 | 学习记录继续绑定原学生账号 |
| 审计记录 | 记录操作人、时间、合并对象、影响范围 |

### 13.4 确认弹窗


学校合并：

```text

说明：确认将学校「<school_name>」合并到当前学校？将影响 <student_count> 名学生、<teacher_count> 名老师。

```

## 18. 验收标准

| 场景 | 验收标准 |
| --- | --- |
| UI 复刻 | 11 张 Web 设计图在 Mobile 中均有对应页面或弹窗，视觉、字段、文案、按钮、状态完整对齐 |
| 学生/家庭能力 | Mobile v1.0 所有验收项继续通过 |
| 老师登录 | 手机号 + 密码或验证码登录成功后进入对应后台 |
| 老师注册 | 老师填写注册信息、验证码和协议后可创建账号 |
| 管理员路由 | 管理员默认进入老师账号管理，可访问老师账号、学生账号、学校信息管理 |
| 教学老师路由 | 普通老师默认进入学生账号管理，不能访问管理员页面 |
| 学生编号生成 | 单个创建成功后生成数字 ID 和初始密码 |
| 批量导入 | CSV/Excel 导入可创建多个学生账号，并展示成功/失败反馈 |
| 生成结果 | 展示学校、班级、数字 ID、初始密码，并支持导出和打印 |
| 管理员学生管理 | 可查看全校学生、班级筛选、处理忘记密码、重置密码、导出、打印 |
| 教学老师学生管理 | 只展示授权范围内学生和忘记密码申请 |
| 老师账号管理 | 管理员可新增老师、查看老师列表、合并老师账号/信息 |
| 学校信息管理 | 管理员可查看学校统计、班级列表、待合并记录和合并记录 |
| 学校/班级合并 | 合并前二次确认；合并后学生 ID 不变，权限迁移，审计记录可查 |
| 权限拦截 | 无权限角色看不到入口；直接访问也被服务端拒绝 |
| 导出安全 | 导出文件只包含权限范围内数据，服务端保留审计，不在日志打印密码 |
| 打印安全 | 账号卡不包含学生真实姓名、完整学号、身份证号 |
| 移动端适配 | 手机竖屏无文字截断、按钮遮挡、表格无法操作等问题 |

## 20. 风险与注意事项

| 风险 | 处理建议 |
| --- | --- |
| Web UI 直接搬到手机导致表格拥挤 | 平板/横屏完整复刻，手机竖屏用横向滚动或字段卡片，但不得删字段 |
| 导出文件包含密码 | 严格按权限导出，记录审计，不写入日志，不长期缓存 |
| 老师误导出全校数据 | 所有导出和打印由服务端按角色过滤，前端不拼权限 |
| 学校合并影响范围大 | 必须二次确认并展示影响学生/老师数量 |
| 老师账号合并误操作 | 源账号和目标账号必须明确展示，禁止相同账号合并 |
| Mobile 与 Web 规则漂移 | 以 Web v1.1 API、组件文案和截图为基线；移动端只做原生适配 |
| 原生打印能力平台差异 | iOS/Android 均需提供至少一种可用交付方式：打印、PDF 预览或分享 |
| 初始密码暴露 | 仅在生成、重置、导出、打印的必要场景展示；离开页面后不保留明文 |
