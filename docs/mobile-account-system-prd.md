# 欧波开心学 Mobile 端账号体系与学习入口 PRD

版本：v1.0  
日期：2026-06-24  
适用项目：`fun-talk-mobile`  
Web 基线：`../../PRD-v1.1.md`（Web 端 v1.1 阶段开发已完成）  
后续 Web 规划：`../../PRD-v2.0.md`

## 1. 文档目的

Web 端已完成 `PRD-v1.1.md` 阶段的账号体系开发。Mobile v1.0 的目标是**复刻 Web 端学生/家庭相关的设计与业务逻辑**，在 App 内原生实现学校学生登录、家庭登录、统一个人中心及账号绑定能力。

Mobile v1.0 **不包含**教师注册/登录、后台管理、学生编号生成、ID 导出、学校信息合并等 Web 端教师/管理员能力。这些功能继续由 Web 端承接。

Mobile 端的核心变化是：学校学生登录从「学校 + 班级 + 姓名 + 密码」切换为「学生数字 ID + 密码」，并在 App 内补齐家庭登录、统一个人中心、家庭账号绑定等能力。

## 2. 产品定位

| 端 | 主要用户 | 定位 |
| --- | --- | --- |
| Web 端 | 老师、管理员、学生、家长 | 账号创建、后台管理、数据管理、学生/家庭登录（v1.1 已完成） |
| Mobile 端 v1.0 | 学生、家长 | 复刻 Web 学生/家庭登录与个人中心逻辑，承接课程学习入口 |

Mobile 端不采集学生真实姓名、真实完整学号、身份证号等私人信息。学生身份仅由系统生成的数字 ID、学生自设昵称和家庭绑定关系识别。

## 7. 功能需求

### 7.1 登录页

#### 7.1.1 页面布局

Mobile 端采用单屏卡片式登录，复刻 Web 统一登录页逻辑：

| 区域 | 说明 |
| --- | --- |
| 顶部 | 欧波开心学 Logo、小狐狸品牌元素 |
| 主卡片 | 两个 Tab：「家庭账号」「学校账号」 |
| 底部 | 协议勾选 |

在手机竖屏中不采用 Web 左右双栏，而是使用 Tab 切换。

Mobile v1.0 **不提供**老师注册/登录入口。

#### 7.1.2 家庭账号登录

| 功能 | 规则 |
| --- | --- |
| 微信登录 | 使用原生微信 SDK，首次登录自动创建家庭账号 |
| 手机号 | 必填，手机号格式校验 |
| 验证码 | 支持获取短信验证码，60 秒倒计时 |
| 密码 | 必填；首次登录时设置密码 |
| 登录/注册 | 手机号首次登录自动创建家庭账号 |

#### 7.1.3 学校账号登录

| 字段 | 规则 |
| --- | --- |
| 数字 ID | 必填，仅支持老师生成的学生数字 ID |
| 密码 | 必填 |
| 登录按钮 | 登录成功后进入课程首页 |
| 忘记密码 | 通知管理员和教学老师，由老师在 Web 后台重置 |

Mobile 端不支持昵称登录。昵称仅用于展示和识别。

#### 7.1.4 协议勾选

登录前必须显式勾选：

- 《用户协议》
- 《隐私协议》
- 《儿童隐私政策》

未勾选时点击登录，提示「请先阅读并同意相关协议」。

### 7.2 学校账号忘记密码

流程（对齐 Web v1.1）：

1. 学生在学校账号登录页点击「忘记密码」。
2. App 弹出输入框，要求输入数字 ID。
3. 系统向该学生所属班级的管理员和教学老师发送短信/站内通知。
4. App 提示「已通知老师，请等待老师重置密码」。
5. 老师在 Web 后台重置密码。
6. 学生使用数字 ID + 新密码重新登录。

### 7.3 统一个人中心

学生账号与家庭账号共用同一「个人中心」页面，根据当前登录身份渲染对应模块。逻辑对齐 Web v1.1 统一个人中心。

#### 7.3.1 入口

课程首页右上角或工具栏中增加「个人中心」入口；学校账号与家庭账号登录后均进入同一页面。

#### 7.3.2 学校账号视图

| 模块 | 字段 / 功能 |
| --- | --- |
| 基础信息 | 数字 ID（只读）、昵称、绑定手机号（可选） |
| 昵称校验 | 不能只有数字，不能重复 |
| 隐私约束 | 不展示学校/班级、真实姓名、真实完整学号 |
| 修改密码 | 新密码 + 确认密码 |
| 绑定家庭账号 | 家庭手机号 + 验证码 + 学校数字编号 |

修改昵称流程：

1. 学生输入昵称。
2. App 本地校验不能纯数字。
3. 服务端校验昵称唯一。
4. 保存成功后课程首页和个人中心同步更新昵称。

#### 7.3.3 家庭账号视图

对齐 Web v1.1 家庭视图：

| 模块 | 字段 / 功能 |
| --- | --- |
| 家庭账号 | 微信绑定状态、手机号脱敏信息、修改密码、协议入口 |
| 我的孩子 | 已绑定孩子列表，每个孩子展示数字 ID、昵称、绑定时间 |
| 孩子操作 | 查看学习数据（v1.0 可先跳转占位页）、解除绑定 |
| 绑定新 ID | 输入学生数字 ID，确认后绑定 |
| 退出登录 | 清理本地登录态 |

### 7.4 学校账号与家庭账号绑定

对齐 Web v1.1 多孩子绑定规则。

#### 7.4.1 绑定入口

| 当前登录身份 | 入口 |
| --- | --- |
| 学生学校账号 | 统一个人中心（学生视图）- 绑定家庭账号 |
| 家庭账号 | 统一个人中心（家庭视图）- 绑定新的学生 ID |

#### 7.4.2 绑定规则

| 规则 | 说明 |
| --- | --- |
| 绑定凭证 | 输入学生数字 ID |
| 昵称验证 | 不需要 |
| 学生密码验证 | 不需要 |
| 一个家庭账号可绑定多个学生 | 支持兄弟姐妹场景 |
| 一个学生默认只能绑定一个家庭账号 | 避免数据归属混乱 |
| 授权状态 | 绑定后默认互相授权 |

绑定后，学校和家长都可以查看该学生的学校与家庭学习记录。

#### 7.4.3 绑定确认

输入 ID 后必须展示确认弹窗（对齐 Web v1.1）：

```text
确认绑定该学生账号？
数字 ID：83920475
昵称：小树苗25 / 未设置
绑定后，学校和家庭账号都可以查看该学生的全部学习记录。
```

确认后完成绑定。

#### 7.4.4 异常提示

| 异常 | 提示 |
| --- | --- |
| ID 不存在 | 未找到该学生数字 ID，请确认后重试 |
| ID 已绑定其他家庭账号 | 该学生 ID 已绑定家庭账号，如需调整请联系老师 |
| 账号状态异常 | 该学生账号暂不可绑定，请联系老师 |

#### 7.4.5 解除绑定

家长可在统一个人中心（家庭视图）对已绑定孩子发起解除绑定。

## 9. API 需求

Mobile v1.0 复用 Web v1.1 已实现的 API，App 端只做端侧适配。

### 9.1 登录相关

| API 能力 | 说明 |
| --- | --- |
| 家庭微信登录 | 微信 code 换取家庭账号 session |
| 家庭手机号登录/注册 | 手机号 + 验证码 + 密码 |
| 学校数字 ID 登录 | digital_id + password |
| 忘记密码通知 | digital_id 触发老师通知 |
| 协议版本记录 | 登录时记录用户同意的协议版本 |

### 9.2 个人中心相关

| API 能力 | 说明 |
| --- | --- |
| 获取当前账号资料 | 返回账号类型、数字 ID、昵称、绑定状态 |
| 修改昵称 | 校验格式和唯一性 |
| 修改密码 | 更新当前账号密码 |
| 绑定学生 ID | 家庭账号输入 digital_id 建立绑定 |
| 获取绑定关系 | 返回家庭账号绑定的学生列表（支持多个） |
| 解除绑定 | 家庭账号解除与指定学生的绑定 |

## 10. 本地存储与登录态

| 项目 | 要求 |
| --- | --- |
| session token | 继续使用现有 `src/lib/auth` 持久化机制 |
| 账号类型 | 登录态需包含 `school_student` / `home_account` 等类型 |
| 当前孩子 | 多孩子绑定后，本地保存当前选中 student_id |
| 协议同意 | 服务端保存为准，必要时本地缓存已读状态 |

## 11. 隐私与安全

| 要求 | 说明 |
| --- | --- |
| 不采集学生私人信息 | 不采集姓名、身份证、真实完整学号 |
| 不支持昵称登录 | 降低撞库和猜测风险 |
| 绑定前明确告知 | 绑定后默认互相查看学习记录 |
| 密码安全 | App 不明文持久化密码 |
| Token 安全 | 原生端使用 SecureStore 等安全存储 |
| 越权保护 | App 端不信任本地角色，所有数据由服务端鉴权 |

## 12. 验收标准

| 场景 | 验收标准 |
| --- | --- |
| 学校登录 | 学生使用数字 ID + 密码登录成功 |
| 旧字段移除 | 学校登录不再要求学校名、班级、姓名 |
| 昵称登录 | 输入昵称不能作为登录账号 |
| 家庭登录 | 微信或手机号登录后进入课程首页，可进入统一个人中心（家庭视图） |
| 统一个人中心 | 学生/家庭登录后进入同一页面，按身份展示对应模块 |
| 协议勾选 | 未勾选协议时不能登录 |
| 忘记密码 | 输入数字 ID 后可通知老师重置 |
| 统一个人中心（学生视图） | 可查看数字 ID、设置昵称、修改密码、绑定家庭账号 |
| 统一个人中心（家庭视图） | 可查看家庭账号信息、已绑定孩子列表 |
| 多孩子绑定 | 一个家庭账号可绑定多个学生数字 ID |
| 绑定确认 | 输入学生 ID 后展示确认信息，确认后才绑定 |
| 解除绑定 | 家长可解除与指定学生的绑定 |
| 隐私展示 | 统一个人中心不展示学校名、班级名、真实姓名、真实完整学号 |
| 默认授权 | 绑定后学校数据和家庭数据默认互相可见 |
| 无教师入口 | 登录页不提供老师注册/登录入口 |

## 14. 风险与注意事项

| 风险 | 处理建议 |
| --- | --- |
| 仅输入 ID 即可绑定存在误绑风险 | 必须增加确认弹窗和清晰授权说明 |
| Mobile 与 Web 字段不一致 | 以 Web v1.1 账号模型为准，Mobile 只做端侧适配 |

| 协议勾选改动影响转化 | 文案保持简短，协议链接清晰可点 |
| 多孩子绑定误绑 | 输入 ID 后必须展示确认弹窗，对齐 Web v1.1 异常提示 |
