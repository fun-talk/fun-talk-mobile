## 1. 文档目的

Web 端新版登录注册与后台设计需要同步影响 `fun-talk-mobile`。Mobile 端不直接复刻完整 Web 后台，而是承接同一套账号、隐私、绑定和学习数据逻辑，重点服务学生与家庭场景。

Mobile 端的核心变化是：学校学生登录从「学校 + 班级 + 姓名 + 密码」切换为「学生数字 ID + 密码」，并在 App 内补齐家庭登录、统一个人中心、家庭账号绑定、学习数据入口等能力。

## 2. 产品定位

| 端 | 主要用户 | 定位 |
| --- | --- | --- |
| Web 端 | 老师、管理员、学生、家长 | 账号创建、后台管理、数据管理、学生/家庭登录 |
| Mobile 端 | 学生、家长 | 课程学习、家庭登录、学校 ID 登录、个人中心、数据查看 |

Mobile 端不采集学生真实姓名、真实完整学号、身份证号等私人信息。学生身份仅由系统生成的数字 ID、学生自设昵称和家庭绑定关系识别。

| 当前能力 | 现状 | 新版要求 |
| --- | --- | --- |

## 7. 功能需求

### 7.1 登录页

#### 7.1.1 页面布局

Mobile 端采用单屏卡片式登录：

| 区域 | 说明 |
| --- | --- |
| 顶部 | 欧波开心学 Logo、小狐狸品牌元素 |
| 主卡片 | 两个 Tab：「家庭账号」「学校账号」 |
| 底部 | 协议勾选、老师入口 |

在手机竖屏中不采用 Web 左右双栏，而是使用 Tab 切换或上下分段。

#### 7.1.2 家庭账号登录

| 功能 | 规则 |
| --- | --- |
| 微信登录 | 使用原生微信 SDK，首次登录自动创建家庭账号 |
| 手机号 | 必填，手机号格式校验 |
| 验证码 | 支持获取短信验证码，60 秒倒计时 |
| 密码 | 必填；首次登录时设置密码 |
| 登录/注册 | 手机号首次登录自动创建家庭账号 |

#### 7.1.3 学校账号登录

| 字段 | 规则 |
| --- | --- |
| 数字 ID | 必填，仅支持老师生成的学生数字 ID |
| 密码 | 必填 |
| 登录按钮 | 登录成功后进入课程首页 |
| 忘记密码 | 通知管理员和教学老师，由老师重置 |

Mobile 端不支持昵称登录。昵称仅用于展示和识别。

#### 7.1.4 协议勾选

登录前必须显式勾选：

- 《用户协议》
- 《隐私协议》
- 《儿童隐私政策》

未勾选时点击登录，提示「请先阅读并同意相关协议」。

### 7.2 学校账号忘记密码

流程：

1. 学生在学校账号登录页点击「忘记密码」。
2. App 弹出输入框，要求输入数字 ID。
3. 系统向该学生所属班级的管理员和教学老师发送短信/站内通知。
4. App 提示「已通知老师，请等待老师重置密码」。
5. 老师在 Web 后台重置密码。
6. 学生使用数字 ID + 新密码重新登录。

### 7.3 统一个人中心

学生账号与家庭账号共用同一「个人中心」页面，根据当前登录身份渲染对应模块。

#### 7.3.1 入口

课程首页右上角或工具栏中增加「个人中心」入口；学校账号与家庭账号登录后均进入同一页面。

#### 7.3.2 学校账号视图

| 模块 | 字段 / 功能 |
| --- | --- |
| 基础信息 | 数字 ID（只读）、昵称 |
| 昵称校验 | 不能只有数字，不能重复 |
| 隐私约束 | 不展示学校/班级、真实姓名、真实完整学号 |
| 修改密码 | 新密码 + 确认密码 |
| 绑定家庭账号 | 家庭手机号 + 验证码 + 学校数字编号 |

修改昵称流程：

1. 学生输入昵称。
2. App 本地校验不能纯数字。
3. 服务端校验昵称唯一。
4. 保存成功后课程首页和个人中心同步更新昵称。

建议增加旧密码校验，但 v1.0 可按服务端能力决定。

#### 7.3.3 家庭账号视图

v1.0 先实现基础版本。

| 模块 | 字段 / 功能 |
| --- | --- |
| 家庭账号 | 微信绑定状态、手机号脱敏信息、修改密码 |
| 已绑定学生 | 展示一个已绑定学生数字 ID 和昵称 |
| 绑定学校账号 | 输入学生数字 ID |
| 查看学习数据 | 进入该学生学习数据入口，v1.0 可先跳转占位页 |
| 退出登录 | 清理本地登录态 |

### 7.4 学校账号与家庭账号绑定

#### 7.4.1 绑定入口

| 当前登录身份 | 入口 |
| --- | --- |
| 学生学校账号 | 统一个人中心（学生视图）- 绑定家庭账号 |
| 家庭账号 | 统一个人中心（家庭视图）- 绑定学校账号 / 我的孩子 |

#### 7.4.2 绑定规则

| 规则 | 说明 |
| --- | --- |
| 绑定凭证 | 输入学生数字 ID |
| 昵称验证 | 不需要 |
| 学生密码验证 | 不需要 |
| v1.0 绑定数量 | 单绑定制 |
| 授权状态 | 绑定后默认互相绑定 |

绑定后，学校和家长都可以查看该学生的学校与家庭学习记录。

#### 7.4.3 绑定确认

由于仅输入 ID 即可绑定，App 需要增加确认弹窗：

```text
确认绑定该学生账号？
数字 ID：83920475
昵称：小树苗25 / 未设置
绑定后，学校和家庭账号都可以查看该学生的全部学习记录。
```

确认后完成绑定。

### 7.5 老师入口

Mobile 端登录页保留「老师注册 / 登录」入口，但不实现原生后台。

点击后：

| 场景 | 行为 |
| --- | --- |
| iOS/Android | 打开系统浏览器或 App 内 WebView，进入 Web 老师注册/登录页 |
| Mobile Web | 直接跳转 Web 老师注册/登录页 |

老师注册、学生编号生成、管理员后台、教学老师后台继续由 Web PRD 承接。

## 11. 本地存储与登录态

| 项目 | 要求 |
| --- | --- |
| session token | 继续使用现有 `src/lib/auth` 持久化机制 |
| 账号类型 | 登录态需包含 `school_student` / `home_account` / `teacher` 等类型 |
| 当前孩子 | v1.1 多孩子绑定后，本地保存当前选中 student_id |
| 协议同意 | 服务端保存为准，必要时本地缓存已读状态 |

## 12. 隐私与安全

| 要求 | 说明 |
| --- | --- |
| 不采集学生私人信息 | 不采集姓名、身份证、真实完整学号 |
| 不支持昵称登录 | 降低撞库和猜测风险 |
| 绑定前明确告知 | 绑定后默认互相查看学习记录 |
| 密码安全 | App 不明文持久化密码 |
| Token 安全 | 原生端使用 SecureStore 等安全存储 |
| 越权保护 | App 端不信任本地角色，所有数据由服务端鉴权 |

## 13. 验收标准

| 场景 | 验收标准 |
| --- | --- |
| 学校登录 | 学生使用数字 ID + 密码登录成功 |
| 旧字段移除 | 学校登录不再要求学校名、班级、姓名 |
| 昵称登录 | 输入昵称不能作为登录账号 |
| 家庭登录 | 微信或手机号登录后进入课程首页，可进入统一个人中心（家庭视图） |
| 统一个人中心 | 学生/家庭登录后进入同一页面，按身份展示对应模块 |
| 协议勾选 | 未勾选协议时不能登录 |
| 忘记密码 | 输入数字 ID 后可通知老师重置 |
| 统一个人中心（学生视图） | 可查看数字 ID、设置昵称、修改密码 |
| 统一个人中心（家庭视图） | 可查看家庭账号信息、绑定/管理学生 ID |
| 隐私展示 | 统一个人中心不展示学校名、班级名、真实姓名、真实完整学号 |
| 绑定学生 ID | 家庭账号输入学生数字 ID 后可完成绑定 |
| 默认授权 | 绑定后学校数据和家庭数据默认互相可见 |
| 老师入口 | 点击老师注册/登录后跳转 Web 管理入口 |

## 15. 风险与注意事项

| 风险 | 处理建议 |
| --- | --- |
| 仅输入 ID 即可绑定存在误绑风险 | 必须增加确认弹窗和清晰授权说明 |
| Mobile 与 Web 字段不一致 | 以 Web PRD 的账号模型为准，Mobile 只做端侧适配 |
| 老师误以为 App 有后台 | 登录页老师入口文案注明「前往 Web 后台」 |
| 旧学校登录数据迁移 | 需要后端支持旧账号向数字 ID 账号迁移或停用旧入口 |
| 协议勾选改动影响转化 | 文案保持简短，协议链接清晰可点 |
